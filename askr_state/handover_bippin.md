## Decisions
- Moved path constants to module level in stop.py rather than using real filesystem save/restore in tests — Matches existing pattern in lifecycle.py; cleaner and more testable than filesystem mocking
- Patched imports at source module (askr.session.lifecycle) rather than at call site — Imports happen at function definition time; patching at source ensures mocks are in place before function executes

## Failed Approaches
- Patching paths as local variables inside the function using mock.patch context managers — Local variables cannot be patched; they exist only at runtime inside the function scope
- Using real filesystem paths with save/restore in tests — Fragile and slow; module-level constants with patching is cleaner
