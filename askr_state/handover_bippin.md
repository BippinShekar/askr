## Decisions
- Permission gate holds queued tasks instead of auto-injecting them when dangerous permissions are detected — Prevents silent injection of tasks into unrestricted sessions; requires explicit user approval via IDE popup before proceeding
- Spam-flagged jobs are deferred to end of session instead of retried inline — Reduces session churn and allows other jobs to complete; spam-flagged jobs can be resubmitted in a fresh session with different fingerprint
- Resume portfolio URL is sourced from application_prefill.answers.portfolio_url, not qa_bank.portfolio_url — qa_bank is deprecated; application_prefill is the canonical source for user-provided application data

## Failed Approaches
- Inline retry of spam-flagged Ashby applications with scroll-and-resubmit — Browser fingerprinting persists across retries in same session; spam_warning recurs immediately, wasting session time
- Using JSONL idle heuristic to trigger companion sessions — Idle detection is unreliable; switched to Stop hook signal for deterministic session lifecycle
